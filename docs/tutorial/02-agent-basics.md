# AI Agent 基础知识（零基础版）

本教程用**通俗语言**解释什么是 AI Agent、它由哪些部分组成，以及在本「智能广告投放」项目里 Agent 能帮你做什么。不要求你事先会写代码，文末的「伪代码」仅帮助建立直觉。

---

## 1. 什么是 AI Agent（通俗解释）

你可以把 **AI Agent（智能体）** 想象成一个「会动脑、会动手的小助手」：

- **动脑**：它背后接了一个大语言模型（类似 ChatGPT），能理解你的话、分析情况、做推理。
- **动手**：它不仅「说完就算了」，还可以按设计去**调用工具**——例如查数据库、调广告平台 API、执行一段计算。

和普通「一问一答的聊天机器人」相比，Agent 更强调 **目标导向**：为了完成一个任务，它可能**多步思考**、**多次调用工具**、**记住中间结果**，直到任务结束或达到停止条件。

一句话：**Agent = 大模型 + 明确目标 + 可选记忆 + 可调用的工具 + 执行策略（规划/循环）**。

---

## 2. Agent 的四大要素

专业讨论里常提到四个关键词：**LLM、Memory、Tools、Planning**。下面用生活化例子说明。

### 2.1 LLM（大语言模型）

**作用**：负责「理解和生成自然语言」，以及在一定程度上做推理。

- 输入：任务描述、当前状态、工具返回的数据等。
- 输出：下一步要做什么的文字说明、结构化决策（如 JSON）、或给用户的报告。

没有 LLM，系统也可以靠固定规则运行，但灵活性和对复杂场景的描述能力会差很多。

### 2.2 Memory（记忆）

**作用**：让 Agent **记住**对话历史、任务中间结果、用户偏好等。

常见类型：

- **短期记忆**：当前这次任务里的上下文（多轮对话、上一步输出）。
- **长期记忆**：向量数据库、知识库等（本项目可扩展，入门阶段可先理解为「状态字典里带着走」）。

记忆太差会导致「说完就忘」；记忆太多会带来噪音和成本，需要设计取舍。

### 2.3 Tools（工具）

**作用**：把 LLM 的「想法」落到**真实世界**：查数、改数、发请求、跑代码等。

例如广告场景里的工具可能是：

- 查询某 Campaign 的曝光、点击、转化。
- 下调或上调出价。
- 生成一版新文案并提交审核（视平台能力而定）。

工具通常以**函数/API**形式注册给框架，由 Agent 在合适时机调用。

### 2.4 Planning（规划）

**作用**：决定**先做哪步、后做哪步**，是否**循环重试**，何时**结束**。

- 简单规划：固定流水线——先监控，再分析受众，再改创意……
- 复杂规划：由 LLM 根据当前状态**动态选择**下一步（路由到不同子 Agent）。

本项目的 Python 版使用 **LangGraph** 做图状编排，就是一种显式的规划结构。

## 3. 单 Agent vs 多 Agent

### 3.1 单 Agent

一个 Agent 包揽所有事：看数据、想策略、调工具、写总结。

- **优点**：结构简单，调试相对集中。
- **缺点**：角色混杂时提示词（Prompt）容易臃肿；复杂业务难以分工。

### 3.2 多 Agent

多个 Agent **各司其职**，例如：

- **监控 Agent**：只看指标和告警。
- **受众 Agent**：只做人群分析。
- **创意 Agent**：只负责素材与文案变体。
- **竞价 Agent**：只处理出价逻辑。
- **优化 Agent**：汇总各方输出，给出可执行方案。

再由 **编排层（Orchestrator / Supervisor）** 决定调用顺序、并行或条件分支。

- **优点**：边界清晰，便于扩展和测试。
- **缺点**：需要设计**状态传递**和**冲突解决**（两个 Agent 意见相反怎么办）。

**本项目是典型的多 Agent + Supervisor 编排**（详见 LangGraph 教程与源码 `python/src/orchestrator/supervisor.py`）。

## 4. Agent 工作流程图（文字描述）

下面用纯文字描述一个**常见的广告优化闭环**（与项目理念一致，细节以代码为准）：

```
开始
  ↓
[监控 Agent] 读取 Campaign 指标，识别异常（如 CTR 骤降、超预算）
  ↓
[受众 Agent] 分析人群维度（年龄、设备、地域等），输出洞察
  ↓
[创意 Agent] 根据洞察建议新标题/素材方向或变体
  ↓
[竞价 Agent] 根据效果与目标调整出价策略
  ↓
[优化 Agent] 汇总上述结果，生成「优化动作列表」和「预算分配建议」
  ↓
判断：是否还有未处理告警？是否未超过最大迭代次数？
  ├─ 是 → 回到 [监控 Agent]（形成循环）
  └─ 否 → 结束，输出报告
```

这种「流水线 + 条件回路」在 LangGraph 里通常用 **边（Edge）** 和 **条件边（Conditional Edge）** 表达。

## 5. 常见 Agent 框架简介

### 5.1 LangGraph

- **特点**：用**有向图**描述 Agent 工作流；节点是函数或 Agent；边表示流转；支持**条件分支**与**循环**。
- **适合**：你希望流程**可可视化、可控制、可回放**，而不是完全黑盒。
- **本项目**：Python 版 Supervisor 使用 LangGraph 编排多 Agent（可选依赖，缺失时回退顺序执行）。

### 5.2 CrewAI

- **特点**：强调「角色（Role）+ 目标 + 背景故事」式的**团队协作**隐喻，API 偏高层。
- **适合**：快速搭原型、演示多角色对话式协作。

### 5.3 AutoGen（Microsoft）

- **特点**：多 Agent **对话协商**；可与代码执行、工具调用结合。
- **适合**：研究型、对话驱动、需要多轮讨论的自动化任务。

**对比小结**：若你重视**工程可控的流程图**，LangGraph 更贴切；若重视**角色扮演与对话式协作**，可了解 CrewAI / AutoGen。本项目选型以 **LangGraph** 为主。

## 6. 广告场景中 Agent 能做什么

下列能力在真实产品中常由 Agent 或「Agent + 规则引擎」共同完成，**教学项目可能以模拟数据演示**：

| 场景 | Agent 可能做的事 |
|------|------------------|
| 监控 | 自动发现异常指标，生成自然语言告警说明 |
| 诊断 | 结合细分维度解释「为什么」变差 |
| 创意 | 批量生成 A/B 文案，并给出测试假设 |
| 竞价 | 在约束下建议出价区间或调整幅度 |
| 预算 | 在多 Campaign 间建议预算再分配 |
| 报告 | 把结构化结果转成运营可读日报/周报 |

注意：**线上投放涉及资金与合规**，实际生产环境必须有权限控制、人工审核与审计日志，Agent 输出通常作为**建议**而非自动生效。

## 7. 示例：最简单的 Agent（Python 伪代码）

下面不是可运行完整程序，而是帮助理解「循环：模型思考 → 调工具 → 再思考」。

```python
# ========== 伪代码：极简 ReAct 风格 Agent ==========

def simple_agent(user_goal, max_steps=5):
    memory = []  # 记忆：每步的思考与观察
    for step in range(max_steps):
        # 1) LLM 根据「目标 + 记忆」决定下一步
        plan = llm.chat(
            system="你是一个广告助手，可调用工具。",
            messages=memory + [{"role": "user", "content": user_goal}],
        )

        # 2) 若模型决定结束，直接返回
        if plan.action == "FINISH":
            return plan.final_answer

        # 3) 若模型要调工具
        if plan.action == "TOOL":
            observation = call_tool(plan.tool_name, plan.tool_args)
            memory.append({"thought": plan.thought, "observation": observation})
            continue

        # 4) 其它情况：保守结束，避免死循环
        return plan.text

    return "达到最大步数，未完成。"


def call_tool(name, args):
    if name == "get_campaign_metrics":
        return database.query_metrics(args["campaign_id"])
    return "未知工具"
```

**解读**：

- `memory` 对应 **Memory**。
- `llm.chat` 对应 **LLM**。
- `call_tool` 对应 **Tools**。
- `for step in range(max_steps)` 对应最简单的 **Planning / 停止条件**。

真实框架会把这些封装得更好，并处理流式输出、并行、持久化等。

## 9. 小结

- **Agent** 是在大模型基础上，为完成目标而使用**工具**与**规划**的程序结构。
- **四大要素**：LLM、Memory、Tools、Planning 缺一不可（可简可繁）。
- **多 Agent** 通过 **Supervisor** 编排，适合复杂广告场景的分工。
- **LangGraph** 适合用图表达流水线与循环；**CrewAI / AutoGen** 各有对话与角色协作特色。
- 广告领域 Agent 擅长**建议与自动化辅助**，生产环境需**风控与人工把关**。
